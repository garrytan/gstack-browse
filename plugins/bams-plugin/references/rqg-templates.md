# Release Quality Gate 출력 템플릿

## 출시 판단 보고서
```
## 출시 품질 게이트 판단

### 판단 결과: [GO / NO-GO / CONDITIONAL GO]
### 판단 일시: [YYYY-MM-DD HH:MM]
### 릴리스 버전: [버전]

### 1. 기능 완성도
| 기능 | 상태 | 검증 결과 | 비고 |
|------|------|-----------|------|

### 2. 결함 현황
- Critical: [N건] / Major: [N건] / Minor: [N건]
- 미해결 차단 결함: [목록 또는 "없음"]

### 3. 테스트 결과
- 회귀 테스트: [통과율]
- E2E 테스트: [통과율]
- 성능 테스트: [기준 대비 결과]

### 4. 운영 준비 상태
- 모니터링: [설정 완료 여부]
- 롤백: [준비 완료 여부]
- 문서화: [완료 여부]

### 5. 잔여 리스크 및 대응 계획
### 6. 차단 사유 (NO-GO인 경우)
### 7. 조건 및 기한 (CONDITIONAL GO인 경우)
```

## 출시 후 모니터링 리포트
```
## 출시 후 품질 모니터링

### 릴리스 버전: [버전]
### 출시 시각: [YYYY-MM-DD HH:MM]
### 모니터링 체크포인트: [30분 / 1시간 / 4시간 / 24시간]

| 지표 | 출시 전 기준 | 현재 값 | 변동률 | 상태 |
|------|-------------|---------|--------|------|

### 이상 징후 발견 사항
### 에스컬레이션 이력
### 롤백 필요성 판단: [불필요 / 검토 중 / 실행 결정]
```

## 롤백 준비 상태 체크리스트
```
## 롤백 준비 상태

| 항목 | 상태 | 검증 방법 | 담당 |
|------|------|-----------|------|
| 롤백 스크립트 존재 | [OK/NG] | | |
| 롤백 스크립트 검증 완료 | [OK/NG] | | |
| DB 역방향 호환 | [OK/NG] | | |
| 이전 버전 아티팩트 보존 | [OK/NG] | | |
| 피처 플래그 비활성화 절차 | [OK/NG/N/A] | | |
| 롤백 예상 소요 시간 | [N분] | | |
| 롤백 트리거 조건 정의 | [OK/NG] | | |

### 롤백 실행 절차
### 롤백 후 검증 항목
```
