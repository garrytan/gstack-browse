How it work.

Install, voice, philosophy, troubleshooting

Install

curl -fsSL cavestack.jerkyjesse.com/install | sh

Works on macOS, Linux, Windows (WSL or git bash). Script install skills to ~/.claude/skills/cavestack/. Symlinks them top-level so Claude Code find them as /qa, /ship, rest.

Caveman voice

Every skill speak compressed caveman. Drop articles. Drop filler. Fragments OK. Code, commits, PRs stay normal prose. Levels: /caveman lite, /caveman full, /caveman ultra. Kill per-session: stop caveman.

Philosophy: judgment before code

  • Completeness (P1): no partial work as done.
  • Blast radius (P2): fix at root, not symptom.
  • Pragmatic (P3): ship working, iterate later.
  • DRY (P4): one source of truth.
  • Explicit over clever (P5): no hidden magic.
  • Bias toward action (P6): pick, commit, verify.

Model-agnostic metrics

Savings measured in characters, not tokens. Numbers survive model switch. 2,400-char answer stay 2,400 chars on Opus, Sonnet, Haiku, GPT, Gemini. Token math lie across providers. Character math honest.

What's on disk

~/.claude/skills/cavestack/   # 40 skills
~/.cavestack/projects/<slug>/ # plans, reviews, checkpoints, timeline
~/.cavestack/analytics/       # local-only usage log
~/.cavestack/config.yaml      # prefix, proactive mode, voice

Nothing leave your machine. No telemetry. No calls to cavestack servers. No servers.

Team mode

Install once. Share via ./setup --team. Teammate run same command. Same skills. No vendoring into repo.

Troubleshooting

  • Skill not found: bin/cavestack-relink — rewrite symlinks.
  • Caveman too terse: /caveman lite — keep more prose.
  • Upgrade: /cavestack-upgrade or cd ~/.claude/skills/cavestack && git pull && bun run build.
  • Proactive prompts annoy: bin/cavestack-config set proactive false.
